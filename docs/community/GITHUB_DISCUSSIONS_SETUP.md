## Enable GitHub Discussions

### **Step 1: Navigate to Repository Settings**

1. Go to https://github.com/creator35lwb-web/VerifiMind-PEAS
2. Click **Settings** tab (top right)
3. Scroll down to **Features** section
4. Check the box next to **Discussions**
5. Click **Set up discussions**

### **Step 2: Choose Starting Category**

GitHub will prompt you to create a first discussion. You can:
- **Option A**: Use GitHub's default welcome template (recommended for quick start)
- **Option B**: Skip and create custom welcome post (recommended for professional setup)

**Recommendation**: Choose **Option B** and use the custom welcome post template below.

## Moderation Guidelines

### **Moderation Principles**

**Be proactive, not reactive**:
- ‚úÖ Welcome new users (reply to first posts)
- ‚úÖ Answer questions promptly (within 24 hours)
- ‚úÖ Highlight great contributions (upvote, comment)
- ‚úÖ Move off-topic discussions to appropriate categories

**Be firm but fair**:
- ‚ö†Ô∏è Warn first-time violations (private message)
- ‚ö†Ô∏è Temporarily hide spam or inappropriate content
- üö´ Ban repeat offenders (after 2-3 warnings)

### **Common Moderation Scenarios**

#### **Scenario 1: Off-Topic Post**

**Action**: Move to appropriate category or General

**Template**:
```
Hi @username! Thanks for posting. I've moved this to the [Category] category 
where it's more likely to get relevant responses. Welcome to the community! üéâ
```

#### **Scenario 2: Duplicate Question**

**Action**: Link to existing discussion, close as duplicate

**Template**:
```
Hi @username! Great question. This has been discussed before in [link to discussion]. 
Check out that thread - if you have additional questions, feel free to ask there!
```

#### **Scenario 3: Low-Quality Post**

**Action**: Ask for clarification or more details

**Template**:
```
Hi @username! Thanks for posting. Could you provide more details about [specific aspect]? 
This will help the community provide better answers. See our [guide to asking questions] 
for tips on how to get the best responses.
```

#### **Scenario 4: Spam or Self-Promotion**

**Action**: Hide post, warn user (first time), ban (repeat offense)

**Template** (first warning):
```
Hi @username. We noticed your post is primarily promotional. Our community guidelines 
ask that promotional content be relevant and provide value to the community. 
Please review our [Code of Conduct] before posting again. This post has been hidden.
```

#### **Scenario 5: Rude or Inappropriate Behavior**

**Action**: Hide comment, warn user, escalate to ban if needed

**Template**:
```
Hi @username. Your comment violates our Code of Conduct (specifically [rule]). 
We've hidden this comment. Please review our community guidelines. 
Further violations may result in a ban from the community.
```

### **Response Time Goals**

**Target response times**:
- **Announcements**: Post within 24 hours of major updates
- **Q&A**: Respond within 24 hours (weekdays), 48 hours (weekends)
- **Ideas**: Acknowledge within 48 hours, detailed response within 1 week
- **Show and Tell**: Acknowledge within 48 hours
- **Moderation**: Act within 12 hours for violations
