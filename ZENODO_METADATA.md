## Zenodo Setup Instructions

### Step 1: Connect GitHub to Zenodo

1. Go to https://zenodo.org
2. Log in with your GitHub account (or create Zenodo account and link GitHub)
3. Go to Settings → GitHub
4. Enable the VerifiMind-PEAS repository
5. Zenodo will now automatically create DOIs for new releases

### Step 2: Create GitHub Releases

**For Genesis v2.0**:
```bash
git push origin genesis-v2.0
```

**For VerifiMind-PEAS v1.0**:
```bash
git push origin v1.0.0
```

Then on GitHub:
1. Go to Releases
2. Click "Create a new release"
3. Select the tag (genesis-v2.0 or v1.0.0)
4. Add release title and description from RELEASE_NOTES files
5. Publish release

### Step 3: Zenodo Automatic DOI Creation

Once releases are published on GitHub:
1. Zenodo automatically detects the release
2. Creates a DOI for each release
3. Archives the release permanently
4. Provides DOI badge and citation information

### Step 4: Update Documentation with DOIs

After Zenodo creates DOIs:
1. Update README.md with actual DOI badges
2. Update CITATION.cff with actual DOIs
3. Update release notes with DOI information
4. Commit and push changes

### Step 5: Verify Citations

1. Check that "Cite this repository" button appears on GitHub
2. Verify DOI badges work and link to Zenodo
3. Test citation formats (APA, BibTeX, IEEE)
4. Confirm Zenodo pages display correctly
