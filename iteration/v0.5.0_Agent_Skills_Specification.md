## 3. Lightweight Implementation Options

Given the budget constraint of **zero new infrastructure costs**, here are three implementation approaches for skill documentation endpoints:

### 3.1 Option A: Static JSON Files (Recommended)

**Implementation:**
- Pre-generate JSON documentation for each skill
- Serve as static files from existing HTTP server
- No database, no dynamic generation

**Cost:** $0 additional

**Pros:**
- Zero runtime cost
- Fast response times
- Easy to maintain
- Works with existing infrastructure

**Cons:**
- Manual updates required for changes
- No dynamic content

**Implementation Details:**
```python
# In http_server.py - add static file serving
@app.get("/skills/{skill_name}")
async def get_skill_documentation(skill_name: str):
    """Serve pre-generated skill documentation."""
    skill_file = f"skills/{skill_name}.json"
    if os.path.exists(skill_file):
        with open(skill_file) as f:
            return JSONResponse(json.load(f))
    raise HTTPException(404, "Skill not found")
```

### 3.2 Option B: Dynamic Generation from MCP Schema

**Implementation:**
- Generate documentation dynamically from existing MCP tool schemas
- Reuse the tool definitions already in server.py
- Transform MCP format to Agent Skills format on-the-fly

**Cost:** $0 additional (uses existing code)

**Pros:**
- Always in sync with actual implementation
- Single source of truth
- No duplicate maintenance

**Cons:**
- Slightly slower (runtime transformation)
- More complex code

**Implementation Details:**
```python
def mcp_to_agent_skill(tool_definition: dict) -> dict:
    """Transform MCP tool definition to Agent Skills format."""
    return {
        "id": tool_definition["name"],
        "name": tool_definition["name"].replace("_", " ").title(),
        "description": tool_definition["description"],
        "endpoint": f"/skills/{tool_definition['name']}",
        "input_schema": tool_definition.get("inputSchema", {}),
        "output_schema": tool_definition.get("outputSchema", {}),
        "examples": generate_examples(tool_definition["name"])
    }

@app.get("/skills/{skill_name}")
async def get_skill_documentation(skill_name: str):
    """Generate skill documentation from MCP schema."""
    tool = get_mcp_tool_by_name(skill_name)
    if tool:
        return JSONResponse(mcp_to_agent_skill(tool))
    raise HTTPException(404, "Skill not found")
```

### 3.3 Option C: Markdown Files with YAML Frontmatter

**Implementation:**
- Write documentation as Markdown files
- Parse YAML frontmatter for structured data
- Serve both human-readable and machine-readable formats

**Cost:** $0 additional

**Pros:**
- Human-readable documentation
- Easy to edit and maintain
- Version control friendly
- Can serve as both docs and API

**Cons:**
- Requires Markdown parsing
- Two formats to maintain (MD + JSON response)
